Deploying FluentUI applications

Deployment of FluentUI applications is very similar to deployment of Qt Quick Controls 2 applications.

FluentUI uses the Qbs build system, but at the moment this doesn't support Android and iOS deployments so we added support for QMake.

The easiest way to use FluentUI for Linux desktop projects is to install a Linux distribution package, but if you want to support Android and iOS as well as Windows you should use qmake and take a look at the minimalqmake example in the examples/perproject/minimalqmake.

This is called the "per-project installation".

Create a directory for your project that we call minimalqmake and inside clone FluentUI:

 mkdir minimalqmake
 cd minimalqmake
 git clone -b master https://github.com/zhuzichu520/FluentUI-Pro.git

Now copy the minimalqmake.pro project file and src/ directory from the example.

Material Design icons can either be installed alongside the FluentUI.Controls QML plugin (the default), or be embedded into the resources.

Pass CONFIG+=fluentui_resource_icons to qmake in order to embed icons into the resources.
